# guides/ – технические мини-гайды под курс

В этой папке лежат короткие практические шпаргалки по инструментам, которые мы используем на курсе «Инженерия Искусственного Интеллекта».

Идея:

- не превращать лекции и семинары в справочник по настройке окружения;
- собрать «инфраструктурные» вещи в одном месте;
- дать понятный маршрут: **какой файл открыть, если что-то не работает**.

## Как этим пользоваться

- Если **что-то не запускается** (Python, ноутбук, сервис) – сначала загляни сюда.
- Перед первой работой в репозитории курса имеет смысл один раз пробежаться по:
  - `unix-101.md`
  - `git-101.md`
  - `python-101.md`
  - `uv-101.md`
  - `jupyter-101.md`

Остальные файлы – по мере надобности (CLI, FastAPI, EDA).

## Список доступных гайдов

### Базовая рабочая среда

- **`unix-101.md`** – минимальный Unix/терминал под курс  
  Навигация по папкам, запуск команд, работа с путями, базовые утилиты.  
  Показывает, как запускать Python-скрипты и Jupyter из терминала, в том числе через `uv run`.

- **`git-101.md`** – Git для сдачи ДЗ и проекта  
  Клонирование репозитория, статус/коммиты/пуш, ветки по минимуму.  
  Описывает типовой сценарий: `git clone → uv sync → uv run ...` для репозиториев курса.

- **`python-101.md`** – Python-проект под задачи курса  
  Структура проекта, модули и пакеты, «ручной» путь с `venv` и `pip`, запуск скриптов.  
  Помогает понять, как устроен код внутри репозитория и как его запускать осознанно.  
  Даёт ссылки на более узкие темы: `uv-101`, `cli-101-typer`, `fastapi-101`, `eda-101`.

---

### Окружение и зависимости

- **`uv-101.md`** – менеджер окружений и зависимостей на курсе  
  Наш основной инструмент для работы с Python-проектами:
  - `uv sync` – установка зависимостей по `pyproject.toml`;
  - `uv run ...` – запуск скриптов, ноутбуков и сервисов в правильном окружении;
  - `uv add/remove` – управление зависимостями.  
  Если репозиторий курса «не заводится», начать стоит именно с этого гайда.

---

### Ноутбуки и EDA

- **`jupyter-101.md`** – работа с Jupyter под курс  
  Как запускать Jupyter (в том числе через `uv run jupyter lab`), как выбирать kernel,  
  что значит «нотбук должен проходить `Restart & Run All`», базовые требования к структуре ноутбуков.

- **`eda-101.md`** – экспресс-анализ табличных данных  
  Минимальный набор приёмов для EDA:
  - загрузка данных (Pandas),
  - первичный осмотр (`head/info/describe`),
  - пропуски и дубликаты,
  - простые группировки и графики (Matplotlib/Seaborn),
  - структурированные выводы по данным.  
  Полезно открывать каждый раз, когда начинаешь новую задачу с табличными данными.

---

### CLI и веб-сервисы

- **`cli-101-typer.md`** – командные интерфейсы на Typer  
  Как из набора функций сделать аккуратное CLI-приложение:
  - точка входа `python -m project.cli ...`,
  - команды `train/predict/...`,
  - аргументы и опции,
  - разбиение на подкоманды.  
  Это гайд «как правильно оформить проект под командную строку».

- **`fastapi-101.md`** – минимальный HTTP-сервис для модели  
  Как обернуть модель в FastAPI-сервис:
  - эндпоинты `/health` и `/predict`,
  - Pydantic-модели для запросов/ответов,
  - запуск через `uv run uvicorn ...`,
  - связь с существующим кодом (модели/пайплайны в `core.py`).  
  Нужен, когда по заданию требуется поднять API поверх модели.

## Рекомендуемый порядок чтения

Если ты только начинаешь работать с инфраструктурой курса:

1. `unix-101.md` – чтобы не бояться терминала.
2. `git-101.md` – чтобы уметь забирать и сдавать код.
3. `python-101.md` – чтобы понимать структуру проекта.
4. `uv-101.md` – чтобы правильно настраивать окружение и запускать код.
5. `jupyter-101.md` – чтобы комфортно работать с ноутбуками.
6. `eda-101.md` – чтобы делать нормальный разведочный анализ данных.

По мере развития задач:

- для мини-утилит и пайплайнов → `cli-101-typer.md`;
- для веб-сервисов с моделями → `fastapi-101.md`.
