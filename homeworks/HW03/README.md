# Методический репозиторий курса «Инженерия Искусственного Интеллекта» (далее — ИИИ)

Здесь находится **структура и материалы курса**: описание, план по неделям, материалы лекций и семинаров, методичка по сквозному проекту, правила оценивания и набор инженерных шпаргалок.

Это **не** студенческий репозиторий. Для работы над домашними заданиями и проектом используется отдельный шаблон `aie-student-template`.

---

## 1. Для чего этот репозиторий

Этот репозиторий играет роль «головы» курса:

- фиксирует цели и формат курса;
- задаёт карту из 21 лекции и 21 семинара;
- описывает сквозной мини-проект и требования к нему;
- содержит материалы лекций/семинаров;
- даёт 101-гайды и инженерные заметки, полезные поверх курса.

Студент:

- **читаёт и смотрит материалы** здесь,
- **код и решения** пишет в своём репозитории по шаблону `aie-student-template`.

Преподаватель:

- использует этот репозиторий как основной источник правды по структуре, материалам и правилам курса.

---

## 2. Базовые файлы в корне

В корне находятся ключевые документы:

- [`COURSE-OVERVIEW.md`](./COURSE-OVERVIEW.md) — общее описание курса:
  - цели и результаты обучения;
  - кратко о сквозном проекте;
  - форматы работы и экзамена.

- [`COURSE-SCHEDULE.md`](./COURSE-SCHEDULE.md) — план по неделям:
  - соответствие «Неделя → LNN и SNN»;
  - предварительные названия лекций и семинаров (могут уточняться);
  - связь с папками `lectures/` и `seminars/`.

- [`COURSE-POLICIES.md`](./COURSE-POLICIES.md) — политики и правила:
  - компоненты оценки (домашки, проект, экзамен);
  - общая логика дедлайнов;
  - академическая добросовестность и работа с данными/секретами.

- [`GLOSSARY.md`](./GLOSSARY.md) — справочник терминов курса (роли, данные, модели, сервисы, MLOps, LLM/RAG, безопасность).

- [`HOW-TO-ASK-QUESTIONS.md`](./HOW-TO-ASK-QUESTIONS.md) — как задавать вопросы по курсу (каналы, формат, примеры хороших запросов).

- `README.md` (текущий файл) — навигация по репозиторию и связь с другими репо курса.

## 3. Структура папок

Высокоуровневая структура:

- `lectures/` — материалы лекций;
- `seminars/` — материалы семинаров;
- `project/` — методические материалы по сквозному проекту;
- `guides/` — 101-гайды (Git, Python, *nix, Jupyter и т.п.);
- `engineering-notes/` — инженерные шпаргалки (TDD/BDD, Git LFS, DVC/версионирование данных, MLOps-инструменты и т.д.);
- `evaluation/` — материалы по оцениванию.

### 3.1. Лекции (`lectures/`)

Каждой лекции соответствует своя папка:

- `lectures/L01/`, `lectures/L02/`, …, `lectures/L21/`

Типовой состав для `LNN`:

- `LNN-brief.md` — краткое содержание лекции:
  - о чём лекция;
  - ключевые идеи;
  - связь с проектом и последующими неделями.
- `LNN-slides.pdf` — слайды лекции.
- `LNN-demo/` — (опционально) демонстрационный код и примеры, которые разбираются именно на лекции.

### 3.2. Семинары (`seminars/`)

Каждому семинару соответствует папка:

- `seminars/S01/`, `seminars/S02/`, …, `seminars/S21/`

Типовой состав для `SNN`:

- `SNN-brief.md` — краткое описание:
  - цель семинара;
  - основные шаги;
  - связь с лекцией недели.
- `SNN-homework.md` — формулировка домашнего задания:
  - что нужно сделать;
  - где в студенческом репозитории это должно лежать;
  - критерии зачёта.
- `SNN-demo.ipynb` — демонстрационный ноутбук для семинара.

### 3.3. Сквозной проект (`project/`)

Папка `project/` описывает сквозной мини-проект:

- `overview.md` — идея и структура проекта:
  - какие домены допустимы;
  - ожидания по результату к финалу;
  - связь с неделями курса.
- `examples-topics.md` — примеры тем проектов.
- `faq.md` — ответы на типичные вопросы по проекту.

Структура **реализации** проекта (код, данные, отчёт) описана в студенческом репозитории-шаблоне `aie-student-template`.

### 3.4. Гайды (`guides/`)

Папка `guides/` содержит материалы выравнивания:

- `git-101.md` — базовые команды Git, типичный рабочий цикл, что делать при конфликтах;
- `python-101.md` — минимум по Python для курса;
- `unix-101.md` — работа в *nix-терминале;
- `jupyter-101.md` — как работать с Jupyter-ноутбуками.

Эти файлы помогают быстро «подтянуть» базу и сосредоточиться на инженерии ИИ.

### 3.5. Инженерные заметки (`engineering-notes/`)

Папка `engineering-notes/` — небольшие шпаргалки по полезным практикам и инструментам:

- `tdd-bdd.md` — тестирование, TDD и BDD;
- `git-lfs.md` — когда и как использовать Git LFS;
- `data-versioning.md` — версионирование данных и моделей (DVC, аналоги);
- `ml-experiment-tracking.md` — ведение экспериментов (MLflow и т.п.);
- `infra-notes.md` — минимальный набор инфраструктурных практик (Docker и т.д.);
- `security-notes.md` — дополнение к правилам по безопасности и работе с данными.

Здесь собраны вещи, которые не всегда входят в основную канву курса, но важны для инженера.

### 3.6. Оценивание (`evaluation/`)

Папка `evaluation/` отвечает за прозрачность оценивания:

- `project-evaluation.md` — критерии и шкала оценивания сквозного проекта;
- `homeworks-policy.md` — общие правила по домашним заданиям (дедлайны, просрочки, формат сдачи);
- `seminars-policy.md` — участие в семинарах, активность, формат работы;
- `exam-outline.md` — структура экзамена: что примерно спрашивается и как это связано с лекциями/семинарами.

## 4. Связь с репозиторием студента

Для работы по курсу студент использует **отдельный** репозиторий по шаблону `aie-student-template`, в котором находятся:

- папка `homeworks/` для домашних заданий;
- папка `project/` для сквозного проекта;
- служебные файлы (`README.md`, `SECURITY.md` и т.д.).

Этот meta-репозиторий:

- **описывает**, что нужно делать и какие требования предъявляются;
- **показывает** демо-код, ноутбуки и примерные решения;
- **даёт** 101-гайды и инженерные заметки.

Код и результаты студент хранит **только** в своём репозитории.

---

## 5. Изменения и актуальность информации

Структура репозитория и номера LNN/SNN считаются стабильными.
Названия и детализация тем лекций/семинаров, сроки сдачи и форматы оценивания могут уточняться по ходу курса.

Актуальная информация:

1. оперативные объявления преподавателя (портал/чат курса);
2. файлы `COURSE-OVERVIEW.md`, `COURSE-SCHEDULE.md`, `COURSE-POLICIES.md`;
3. соответствующие файлы в `lectures/`, `seminars/`, `project/` и `evaluation/`.

При расхождении интерпретаций приоритет имеет последняя версия объявлений и этих документов.
