# Unix 101 для курса «Инженерия Искусственного Интеллекта»

Этот файл — короткий гайд по работе в терминале *nix-систем (Linux, macOS WSL и т.п.) **под задачи курса**:

- навигация по файловой системе;
- базовые команды с файлами;
- запуск Python, venv, ноутбуков и скриптов;
- запуск Git-команд из терминала.

Цель: чтобы ты уверенно жил в терминале, не боялся «чёрного окна» и мог повторять то, что показано на семинарах.

---

## 1. Терминал: откуда всё запускается

### 1.1. Как открыть

- **Linux**: «Terminal» / Ctrl+Alt+T.
- **macOS**: `Terminal` (Spotlight: ⌘+Space → «Terminal»).
- **Windows**:
  - если используешь WSL: «Ubuntu»/другая дистрибуция;
  - можно PowerShell, но в Unix-гайде говорим именно про WSL/Linux/macOS.

### 1.2. Где я вообще нахожусь? — `pwd`

```bash
pwd
```

Показывает **текущую папку** (рабочий каталог).
Например:

```text
/home/user/aie-student-2025
```

## 2. Хождение по папкам

### 2.1. Смена папки — `cd`

```bash
cd путь/до/папки
```

Примеры:

```bash
cd ~            # перейти в домашнюю директорию
cd ~/projects   # перейти в папку projects в домашней
cd ..           # подняться на уровень выше
cd ../..        # подняться на два уровня
cd aie-course-meta   # перейти в папку aie-course-meta
```

`~` — это «домашняя директория» пользователя.

### 2.2. Просмотр содержимого — `ls`

```bash
ls          # показать файлы и папки
ls -l       # показать подробно (права, размер, даты)
ls -a       # показать скрытые файлы (в т.ч. .git, .venv и т.п.)
```

Комбинации:

```bash
ls -la      # подробно + скрытые
```

## 3. Файлы и папки: минимум выживания

### 3.1. Создать папку — `mkdir`

```bash
mkdir homeworks
mkdir -p homeworks/HW01
```

Флаг `-p` создаёт промежуточные папки при необходимости.

### 3.2. Копировать — `cp`

```bash
cp source.txt copy.txt                 # копия файла
cp -r source_dir/ target_dir/          # копия папки рекурсивно
```

### 3.3. Переместить/переименовать — `mv`

```bash
mv old_name.py new_name.py             # переименовать
mv file.py homeworks/HW01/             # переместить файл
```

### 3.4. Удалить — `rm` (осторожно!)

```bash
rm file.txt                # удалить файл
rm -r some_directory/      # удалить папку рекурсивно
```

> **Осторожно:** `rm` не спрашивает «точно ли?» и не кладёт в корзину.
> Команда `rm -rf /something` при неправильном применении может вычистить очень много.

## 4. Просмотр содержимого файлов

### 4.1. Быстрый просмотр — `cat`, `head`, `tail`

```bash
cat file.txt       # показать целиком (осторожно с большими файлами)
head file.txt      # первые 10 строк
head -n 20 file.txt    # первые 20 строк
tail file.txt      # последние 10 строк
tail -n 50 file.txt    # последние 50 строк
```

### 4.2. Пейджер — `less`

```bash
less file.txt
```

Управление:

- стрелки / PageUp / PageDown — листать;
- `q` — выход.

## 5. Запуск Python, venv и скриптов

### 5.1. Проверить Python

```bash
python --version
# или
python3 --version
```

В *nix часто используется `python3`, поэтому дальше подразумеваем:

```bash
python3 ...
```

или настраиваем alias, чтобы `python` → `python3`.

### 5.2. Создать и активировать venv

В корне репозитория:

```bash
python3 -m venv .venv
```

Активировать:

```bash
source .venv/bin/activate
```

Теперь в начале строки появится `(.venv)`.

Отключить:

```bash
deactivate
```

### 5.3. Установка пакетов

```bash
pip install -r requirements.txt
# или
pip install numpy pandas scikit-learn
```

> Важно: делать это **после активации** `.venv`, чтобы пакеты ставились внутрь окружения.

### 5.4. Запуск скрипта

```bash
python main.py
python project/src/train_model.py
python -m project.src.service.api
```

Тот же Git, что и в `git-101.md`, но в контексте *nix:


## 8. Права и «Permission denied»

Иногда команда отвечает:

```text
Permission denied
```

Частые причины:

- нет прав на запись в папку;
- файл/скрипт не имеет права на запуск.

### 8.1. Права на файл

Посмотреть:

```bash
ls -l script.sh
```

Права выглядят вроде:

```text
-rwxr-xr-x
```

Сделать файл исполняемым:

```bash
chmod +x script.sh
```

Теперь можно:

```bash
./script.sh
```

> Для задач курса чаще достаточно просто запускать Python командой `python` без изменения прав.

## 9. Перенаправление и пайпы (очень кратко)

### 9.1. Перенаправление в файл

```bash
python script.py > output.txt      # записать вывод в файл (перезаписать)
python script.py >> output.txt     # дописать в конец файла
```

### 9.2. Пайпы

```bash
cat big_file.txt | head -n 20
```

Читаем как: «взять то, что выдал `cat big_file.txt`, и передать это на вход `head -n 20`».

## 10. Мини-чеклист перед семинаром

Перед семинаром, где нужно что-то запускать в *nix-терминале:

1. **Открывается ли терминал?**

   - можешь ли выполнить `pwd`, `ls`, `cd`?

2. **Рабочая директория:**

   - находишься ли ты в корне репозитория (видны `homeworks/`, `project/`, `requirements.txt`)?

3. **Python и venv:**

   - `python3 --version` выдаёт адекватную версию;
   - создано окружение `.venv`;
   - после `source .venv/bin/activate` появляется `(.venv)` и `pip` работает.

4. **Зависимости:**

   - `pip install -r requirements.txt` отрабатывает без критичных ошибок.

5. **Git:**

   - `git status` показывает, что репозиторий существует;
   - ты понимаешь, где `git add / commit / push`.

Если все пункты выполняются, ты в нормальном состоянии для практики.

## 11. Если терминал пугает

Нормально, если поначалу терминал кажется «магией». Стратегия:

1. Наблюдать: что пишет преподаватель на семинаре → что происходит.
2. Повторять команды **вместе** с объяснением.
3. Не бояться спрашивать «что делает эта команда?» — чаще всего ответ помещается в 1–2 предложения.
4. Не запускать непонятные команды, найденные в интернете, **особенно** с `rm`, `chmod`, `sudo`.

С этим минимальным набором команд можно комфортно жить на курсе и постепенно расширять репертуар по мере надобности.
