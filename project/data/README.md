# Данные проекта

## Источник данных

**MovieLens 32M** — открытый датасет, собранный GroupLens Research (Университет Миннесоты).

- **Ссылка:** https://grouplens.org/datasets/movielens/32m/
- **Размер:** 239 МБ (в сжатом виде), 911 МБ (распакованный)
- **Лицензия:** свободная для академического использования

---

## Структура данных

### `data/raw/` — исходные данные (после распаковки)

| Файл | Размер (распакованный) | Описание | Используется |
|------|----------------------|----------|--------------|
| `ratings.csv` | 850 МБ | 32 млн оценок (userId, movieId, rating, timestamp) | Да |
| `movies.csv` | 4 МБ | 87 тыс. фильмов (movieId, title, genres) | Да |
| `tags.csv` | 71 МБ | 2 млн тегов (userId, movieId, tag, timestamp) | Да |
| `links.csv` | 2 МБ | Ссылки на IMDb/TMDB | Нет|
| `README.md` | — | Информация о датасете | Нет |
| `checksums.txt` | — | Контрольные суммы файлов | Нет |

### `data/processed/` — подготовленные данные для моделей

| Файл | Описание | Размер |
|------|----------|--------|
| `movies_processed.csv` | Фильмы с признаками (год, жанры, десятилетие, теги) | ~80 МБ |
| `cb_movieId_to_idx.json` | Маппинг movieId → индекс для CB модели | ~1.5 МБ |
| `cf_movieId_to_idx.json` | Маппинг movieId → индекс для CF модели | ~1 МБ |
| `cf_userId_to_idx.json` | Маппинг userId → индекс для CF модели | ~4 МБ |
| `cf_user_item_matrix.npz` | Разреженная матрица user-item (BM25 вес) | ~150 МБ |
| `neg_cf_movieIds.json` | Негативные фильмы для исключения из рекомендаций | ~250 МБ |

### `data/features/` — признаки для инференса

| Файл | Описание | Размер |
|------|----------|--------|
| `cb_features.csv` | Матрица признаков фильмов (TF-IDF + SVD, 100 компонент) | ~150 МБ |

## Способы получения данных

### Способ 1: Автоматическое скачивание (рекомендуется)

```bash
cd project
python scripts/load_and_prepare_data.py
```

**Что делает скрипт:**
1. Скачивает `ml-32m.zip` с официального сайта в `data/raw/`
2. Распаковывает архив
3. Выполняет предобработку и фильтрацию по порогам
4. Сохраняет обработанные данные в `data/processed/`

### Способ 2: Ручное скачивание

1. Перейти на https://grouplens.org/datasets/movielens/32m/
2. Скачать `ml-32m.zip`
3. Распаковать в `data/raw/ml-32m/`
4. Запустить `python scripts/load_and_prepare_data.py` для обработки
   
Для получения признаков фильмов `cb_features.csv` необходимо также запустить скрипт `python scripts/train_models.py`.

### Способ 3: Использовать готовые артефакты с Hugging Face

```bash
cd project
python scripts/download_models.py
```
Загружает только обработанные данные (`data/processed/`, `data/features/`) и модели, без исходных CSV.

## Почему данные не хранятся в репозитории

| Причина | Объяснение |
|---------|------------|
| **Размер** | Исходные данные ~1 ГБ (распакованные)|
| **Лимиты GitHub** | GitHub не предназначен для хранения больших файлов |
| **Воспроизводимость** | Скрипты загрузки обеспечивают чистоту эксперимента |

## Воспроизводимость

### Для полного воспроизведения эксперимента:

```bash
# 1. Получить данные
python scripts/load_and_prepare_data.py

# 2. Обучить модели
python scripts/train_models.py

# 3. Запустить ноутбуки (опционально)
jupyter lab notebooks/
```

### Только для запуска API (без переобучения):

```bash
# 1. Загрузить готовые модели и данные
python scripts/download_models.py

# 2. Запустить сервис
python scripts/run_api.py
```
