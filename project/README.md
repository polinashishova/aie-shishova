## 1. Паспорт проекта

- **Название проекта:** `Гибридный рекомендательный API сервис фильмов с контент-базированной моделью и колаборативной фильтрацией`
- **Автор:** `Шишова Полина Сергеевна`
- **Группа:** `БФБО-02-23`
- **Контакт:** телеграм: `@waste_magpie`
               почта:    `timewastingnonsense@gmail.com`

- **Краткое описание (2-4 предложения):**  
  
  Проект реализует гибридный сервис рекомендаций фильмов, объединяющий контент-базированный подход (на основе жанров, тегов и года выпуска) и коллаборативную фильтрацию (на основе оценок пользователей).
  Используется открытый датасет, простые content-based модели, модели коллаборативной фильтрации и случайные baseline-модели.
  Результат — REST API, возвращающий для запрошенного ID фильма список похожих, а для ID пользователя — персонализированные рекомендации.

## 2. Структура проекта

Проект организован в следующей структуре:

- `pyproject.toml` – конфигурация проекта: зависимости, инструменты и настройки сборки.
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, эксперименты.
- `src/` – основной код проекта:
  - модуль для работы с данными: `mrh/data.py`;
  - обучение моделей: `mrh/models.py`;
  - эндпоинты, валидация данных и FastAPI приложение: `mrh/api/`;
  - вспомогательные функции: `mrh/utils.py`;
  - работа с переменными окружения: `config.py`.
- `data/` – данные:
  - из-за размера не были закоммичены, но есть скрипт для скачивания `scripts/load_and_prepare_data.py`.
- `configs/` – конфигурационные файлы:
  - `paths.json` - конфигурация путей проекта;
  - `models.json` - конфигурации моделей;
  - `.env.example` - шаблон переменных окружения. 
- `tests/` – тесты.
- `artifacts/` – сохранённые модели, логи (не были закоммичены), артефакты ноутбуков.
- `scripts/` - скрипты для:
  - загрузки и подготовки данных (`load_and_prepare_data.py`);
  - обучения моделей (`train_models.py`);
  - скачивания моделей и данных, необходимых для инференса (`download_models.py`);
  - запуска сервиса (`run_api.py`).
- `dockerfile` - контейнеризация приложения.

Если структура проекта отличается от базовой, опишите это здесь.
В существующем проекте нет файла `requirements.txt`, но есть `pyproject.toml` со всеми зависимостями проекта.
Также дополнительно присутствует папка `scripts/` и `dockerfile`.

## 3. Требования и установка

### 3.1. Требования

- **Python** `>= 3.10`
- **Docker**
- **Системные зависимости** (только для локальной установки на Linux):
  ```bash
  sudo apt-get update
  sudo apt-get install -y build-essential gcc libopenblas-dev libgomp1
  ```
- Для Windows рекомендуется использовать **Docker**

### 3.2. Установка окружения

```bash

cd project

python -m venv .venv

# Активировать окружение:

source .venv/bin/activate

pip install --upgrade pip
```

- Модели загружаются автоматически из Hugging Face Hub при первом запуске (если они не были загружены ранее другим способом (см. п.4 Как запустить проект))

- При локальной установке на Linux требуются системные зависимости для компиляции `numpy`, `scipy` и `implicit`

## 4. Как запустить проект

### 4.1. Запуск через Docker

```bash
cd project

# Сборка образа
docker build -t movie-recsys-hybrid .

# Запуск контейнера
docker run -d -p 8000:8000 --name movie-recsys-hybrid movie-recsys-hybrid

# Остановка контейнера
docker stop movie-recsys-hybrid
```

### 4.2. Запуск API сервиса локально

Сначала необходимо перейти в папку проекта и активировать окружение.

```bash
cd project
# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate
```

Для локального запуска сервиса необходимо иметь следующие файлы:

- `artifacts/models/`: 
                      - `als_model.joblib` (обученная модель коллаборативной фильтрации - `Alternating Least Squares` из библиотеки `implicit`);
                      - `cb_pipeline.joblib` (обученная контент-базированная модель - пайплайн, состоящий из `TF-IDF` векторизатора и `TruncatedSVD` из библиотеки `scikit-learn`);
- `data/`:
   - `processed/`:
                 - `cb_movieId_to_idx.json`  - маппинг для ID фильмов, на которых был обучен контент-базированный пайплан;
                 - `cf_movieId_to_idx.json`  - маппинг для ID фильмов, на которых была обучена ALS модель;
                 - `cf_userId_to_idx.json`   - маппинг для ID пользователей, на которых была обучена ALS модель;
                 - `cf_user_item_matrix.npz` - матрица взаимодействий пользователей и фильмов, на которой была обучена ALS модель;
                 - `neg_cf_movieIds.json`    - фильмы, которым пользователи поставили оценку 3 и ниже, но взаимодействия с которыми отсутствуют в матрице `cf_user_item_matrix.npz`; используется для того, чтобы пользователям данные фильмы не рекомендовались; формат: пользователь - [список ID фильмов, которым пользователь поставил оценки 3 и ниже];
   - `features/`:
                 - `cb_features.csv` - признаки фильмов.

Для того, чтобы их получить, можно воспользоваться одним из следующих способов:

#### 4.2.1. Полная загрузка данных и обучение моделей

```bash
# загрузка данных и их подготовка
python scripts/load_and_prepare_data.py
# обучение моделей и их сохранение
python scripts/train_models.py
```

#### 4.2.2. Скачивание моделей и данных с Hugging Face Hub

```bash
python scripts/download_models.py
```

#### 4.2.3. Запуск ноутбуков (если установлены dev зависимости)

```bash
pip install -e ".[dev]"

python -m ipykernel install --user --name=.venv --display-name "Python (.venv)"

jupyter lab
# или
jupyter notebook
```
Сначала необходимо запустить ноутбук `notebooks/01_eda.ipynb`, затем `notebooks/02_preprocessing_and_experiments.ipynb`.


Первый способ (скачивание данных и обучение моделей) подойдет, если нужны данные в полном объеме.

Второй способ предпочтительнее, если нужно только запустить сервис.

Третий способ рекомендуется использовать, только если интересует экспериментальная часть проекта, при выполнении всех ячеек результат будет таким же, как в первом случае. Если запуск кода экспериментов не обязателен, лучше воспользоваться любым другим из двух выше перечисленных способов.

Все запуски выполняются из папки проекта.


После того, как все необходимые файлы существуют локально, нужно запустить приложение.

```bash
# Запуск сервера (если локально есть необходимые данные и модели)
python scripts/run_api.py 

# С дополнительными параметрами:
python scripts/run_api.py --host 0.0.0.0 --port 8000 --workers 2
```

### 4.3. Проверка работоспособности

Сервис запускается на **порту 8000**.

**Основные эндпоинты:**

| Эндпоинт | Метод | Описание | Пример |
|----------|-------|----------|--------|
| `/` | GET | Информация о сервисе | `curl http://localhost:8000/` |
| `/health` | GET | Проверка здоровья сервиса | `curl http://localhost:8000/health` |
| `/predict` | POST | Получение рекомендаций | см. ниже |
| `/docs` | GET | Swagger UI документация | открыть в браузере |
| `/redoc` | GET | ReDoc документация | альтернативный просмотр описания эндпоинтов |


#### 4.3.1. Проверка работоспособности через curl

**Примеры запросов:**

```bash
# Health check
curl http://localhost:8000/health

# Рекомендации похожих фильмов (cold-start)
curl -X POST http://localhost:8000/predict -H "Content-Type: application/json" -d "{\"movieIds\": [1225, 329], \"k\": 5}"
# Для powershell
iwr -Uri "http://localhost:8000/predict" -Method POST -ContentType "application/json" -Body '{"movieIds": [1225, 329], "k": 5}'

# Персонализированные рекомендации (по пользователю)
curl -X POST http://localhost:8000/predict -H "Content-Type: application/json" -d "{\"userIds\": [214, 154198], \"k\": 10}"
# Для powershell
iwr -Uri "http://localhost:8000/predict" -Method POST -ContentType "application/json" -Body '{"userIds": [214, 154198], "k": 10}'
```

#### 4.3.2. Проверка работоспособности через Swagger UI

1. Открыть Swagger UI документацию `http://localhost:8000/docs`.
2. Для проверки здоровья сервиса в подвкладке `health` нажать на `GET  /health` -> нажать на `Try it out`.
3. Для получения рекомендаций в подвкладке `predict` нажать на `POST  /predict` -> нажать на `Try it out`:
   - для получения контентных рекомендаций в поле `Edit Value | Schema` оставить поля `movieIds` и `k` (или только `movieIds`);
   - для получения персонализированных рекомендаций в поле `Edit Value | Schema` оставить поля `userIds` и `k` (или только `userIds`).

**Пример ответа:**
```json
{
  "recommendations": {
    "1225": [
      {"movieId": 1234, "score": 0.95},
      {"movieId": 5678, "score": 0.89}
    ]
  },
  "model_used": "content_based",
  "timestamp": "2025-05-26T10:30:00Z",
  "request_id": "abc12345"
}
```

## 8. Ограничения и дальнейшая работа

Ограничения:
- В текущей версии в сервисе не предусмотрено добавление в каталог новых фильмов или пользователей, хотя проблема user cold-start решена с помощью контент-базированной модели рекомендаций.
- Также модели обучились однажды на всех доступных данных, и новые оценки пользователей никак не повлияют на работу сервиса до полного переобучения модели.
- Используется только датасет MovieLens, для других источников потребуется адаптация.
- Для контентных рекомендаций в качестве признаков использованы только жанры, год выпуска и теги, т.е. только та информация, которую предоставляет датасет без использования внешних данных о фильмах.
- Сервис не предоставляет объяснения, почему порекомендован тот или иной фильм.
- Система не расчитана на высокие нагрузки.
- Одинаковые запросы от разных пользователей не кэшируются.
- Нет защиты от DoS-атак, в теории сервис можно очень легко убить.
- В проекте сейчас напрямую логируются ID пользователей, более того маппинги userID - index лежат в открытом доступе на Hugging Face Hub, что при использовании реальных ID пользователей было бы недопустимо. Впрочем, данные были взяты из открытого источника, где авторы также указали на то, что ID пользователей, представленные в данном датасете, были аннонимизированы, что позволило на данном этапе воспользоваться этим, упрощая реализацию.

Что можно сделать для развития проекта:
- Реализовать обновление каталога доступных фильмов.
- Поэкспериментировать с признаками фильмов; попробовать использовать информацию, доступную на внешних источниках.
- Добавить возможность частичного переобучения модели коллаборативной фильтрации на новых пользователях.
- Реализовать объяснения предсказаний, например, данный фильм был порекомендован, так как он похож на один из тех, с которыми пользователь взаимодействовал ранее.
- Добавить кэширование запросов и ограничения на количество запросов.
- Придумать механизм защиты настоящих ID пользователей.
